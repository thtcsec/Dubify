Dubify Hackathon Demo Pack (3-minute version)
=============================================

Positioning
-----------
Dubify is not a simple video generator.
Dubify is a Global Marketing Video Studio that turns one product brief into a PixVerse-powered campaign workflow:

Product brief
-> research
-> script
-> scene review
-> PixVerse assets
-> voice/subtitles
-> export

Core story for judges
---------------------
PixVerse generates the visual shots.
Dubify solves everything around it:
- research alignment
- storyboard review
- prompt standardization
- voiceover
- subtitles
- final assembly
- provenance

One-line unfair advantage
-------------------------
Compared with ChatGPT + PixVerse + CapCut, Dubify adds a structured production workflow with human-in-the-loop scene approval, automated post-production, and provenance-backed PixVerse asset verification.

Demo runtime reality
--------------------
If the machine is warm and PixVerse clips are already prepared:
- Research live: around 20-45 seconds
- Approve Script + Scene Review: around 20-30 seconds
- Render with folder clips: around 40-90 seconds
- Total: usually around 1.5 to 2.8 minutes

Recommendation:
- Yes, record a backup screen capture.
- For the live demo, pre-generate PixVerse clips first.
- Keep one final exported MP4 ready in case FFmpeg or TTS stalls.

What to say if a judge asks:
"So is this just ChatGPT + PixVerse + FFmpeg?"
------------------------------------------------
No.
ChatGPT can draft text, PixVerse can generate clips, and FFmpeg can merge files.
Dubify adds the production workflow that connects them:
- research alignment
- scene-level approval
- prompt normalization
- voice and subtitle automation
- export packaging
- provenance

What to say if a judge asks:
"Why not use PixVerse API directly?"
-----------------------------------
Dubify supports API-style provenance logic, but the hackathon environment gives many teams Web credits first.
For demo reliability, we use real PixVerse Web-generated clips and import them through a controlled workflow, which is still fully compliant with the guidebook and reduces live-failure risk.
